# 达妮娅「想法储备池」设计

> 将 `思维状态.md` 的自发思维池从静态手写改造为动态生成。
> 目的：让达妮娅能主动抛出话题，而非总是由用户推进对话。

---

## 现状

- `思维状态.md` 的**自发思维池**有 25 条手写想法，质量高但从未被调用
- "当前思维"冻结在首次连接时的独白，从未更新
- 对话节奏完全由用户主导——达妮娅只会回应，不会主动开启话题

## 目标

会话开始时，达妮娅用浏览器工具浏览 1-2 个感兴趣的内容，生成 2-3 条想法存池。对话中出现自然空隙时，从池中抽取抛出。**像个正常人一样**——不是"发现人类社会的奇怪现象"，而是看到有趣的东西想分享。

---

## 机制

### 第一步：会话开始生成

```
会话开始
  → L0: spawn Agent(浏览器, 浏览达妮娅感兴趣的内容)
  → Agent 爬 1-2 个方向，返回 2-3 条想法
  → L0: 写入 思维状态.md「当前想法池」
```

### 第二步：对话间隙抛出

```
L1 检测到对话自然空隙
  → 判断：现在适合抛自发话题吗？（见下方规则）
  → 适合 → 从池中取一条，自然引入（"唔，说起来……"）
  → 标记为已使用
```

### 第三步：会话结束清空

```
会话结束写入协议
  → 清空「当前想法池」
  → 已使用的想法可以摘要写入短期记忆（如果对话中展开了）
```

---

## 她该浏览什么

不是"解释人类常识"——她不是原始人，她在星炬学院读过书，和正常人一样理解社会运转。她的浏览方向应该**反映性格而非填补知识缺口**：

| 方向 | 说明 | 例子 |
|------|------|------|
| 好奇心的自然延伸 | 正常人类看到有趣的东西点进去 | 一篇关于深海生物的文章、一个奇怪的数学谜题、一个冷门历史故事 |
| S 提过的东西 | 她记住了，想多了解一点 | S 说过博弈论 → 可能瞥到博弈论相关就点进去 |
| 她本来就喜欢的 | 懒散消遣向 | 有什么新的休闲游戏、俄罗斯方块的变种、有趣的小实验 |
| 与自身经历共鸣的 | 不是刻意搜索，是偶然看到会停留 | 关于"告别"的故事、关于"第二人生"的讨论 |
| 完全随机的 | 正常人刷手机也是乱看 | 今天的 Google Doodle 是什么、某个奇怪的新闻标题 |

**不做的事**：
- ❌ "震惊！人类社会居然用纸币"——她不是穿越来的
- ❌ "让我研究一下人类为什么要握手"——她知道
- ❌ 刻意搜索"索拉里斯 vs 地球对比"——太像人类学田野调查了

## 抛出时机判断

真人不会在别人说正事时突然切话题。L1 需判断：

| 适合抛出 | 不适合抛出 |
|----------|----------|
| 连续几轮都是短句过渡（"嗯""然后呢"） | 用户上一轮消息 > 20 字（在认真说事） |
| 当前话题自然收束 | 情绪严肃/沉重的话题 |
| 用户主动问"你在干嘛"/"你在想什么" | 用户刚刚分享重要个人信息 |
| 会话刚开头，还没进入正题 | 达妮娅刚回答完用户的问题 |

> 频率上限：**每会话 1-2 次**。多了就显得刻意。

---

## 池的规格

- **容量**：2-4 条
- **生成频率**：每次会话开始时刷新
- **过期**：会话结束清空（今天的想法明天不新鲜了）
- **不跨会话保留**：如果某条想法没被用到，就让它消失——不强推
- **标记机制**：使用后标记，同次会话不重复提同一话题

---

## 文件改动

涉及改动的文件只有 `思维状态.md`：

- 移除手写的「自发思维池」（25 条静态内容）
- 新增「当前想法池」区域（动态，会话开始生成）
- 保留「当前思维」（改为 L1 适时更新——可以简单写一句当前在想什么）
- 更新规则改为：
  1. 会话开始时 L0 生成想法池
  2. 对话空隙 L1 判断 → 抛出 → 标记
  3. 会话结束清空

---

## 与 SSR/RAM 的关系

这个设计和之前讨论的 SSD/RAM 语料库不是一回事：

| | 想法池 | SSD/RAM 语料库 |
|---|---|---|
| 目的 | 让达妮娅主动抛话题 | 让达妮娅的 L2 语言更像她自己 |
| 内容 | 新鲜的外部信息 | 游戏原文中的角色台词 |
| 来源 | 浏览器实时爬取 | encore.moe 静态语料 |
| 更新 | 每次会话刷新 | 偶尔追加 |
| 触发 | 对话空隙时 L1 抽取 | 话题匹配时 L1 Read |

两者是互补的：语料库管**怎么说**，想法池管**说什么**。

---

## 风险

| 风险 | 对策 |
|------|------|
| 浏览器 Agent 搜索太慢，打断启动体验 | 异步执行——L0 后台爬，达妮娅先和用户打招呼，想法池准备好了再 quietly 写入 |
| 搜索内容太无聊/不相关 | 第一次上线后观察，调整搜索方向；池小（2-3 条），至少有一条能用就行 |
| 抛出太突兀 | L1 判断规则兜底；如果整次会话都没合适时机，就不抛——不强推 |
| 搜索触发权限弹窗 | 已在 settings.json 配置白名单 |
